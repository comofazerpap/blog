Bem-vindo ao GitHub Pages.

Esta página gerador automático é a maneira mais fácil de criar belas páginas para todos os seus projetos. Autor o conteúdo da página aqui, usando GitHub Flavored Markdown, selecione um modelo elaborado por um designer, e publicar. Depois de sua página é gerada, você pode conferir o novo ramo:

` $ Cd your_repo_root / repo_name $ Git fetch origem $ Git checkout gh-pages `

Se você estiver usando o GitHub para Mac, basta sincronizar o repositório e você verá o novo ramo.

Templates Designer

Nós temos trabalhado alguns modelos bonitos para você usar. Vá em frente e continuar a layouts para percorrê-las. Você pode facilmente voltar para editar sua página antes de publicar. Após a publicação de sua página, você pode revisitar o gerador de página e mudar para outro tema. O conteúdo da página será preservado se manteve formato remarcação.

Pelo contrário Unidade Stick?

Se você preferir não usar o gerador automático, aperte um ramo chamado gh-pages ao seu repositório para criar uma página manualmente. Além de apoiar o conteúdo HTML regular, Páginas GitHub apoiar Jekyll, um blog ciente site estático gerador simples, escrito por nosso próprio Tom Preston-Werner. Jekyll torna mais fácil para criar cabeçalhos e rodapés em todo o site, sem ter que copiá-los através de cada página. Ele também oferece suporte blogue inteligente e outros recursos avançados de modelagem.

Autores e colaboradores

Você podemention um nome de usuário GitHub para gerar um link para seu perfil. O elemento resultante irá vincular a GitHub perfil do colaborador. Por exemplo: Em 2007, Chris Wanstrath (defunkt), PJ Hyett (pjhyett) e Tom Preston-Werner (mojombo) fundada GitHub.

Suporte ou Contato

Tendo problemas com páginas? Confira a documentação em http://help.github.com/pages ou contate support@github.com e nós vamos ajudá-lo a resolver o problema.
